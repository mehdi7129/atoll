# vibeisland

## Summary
Vibe Island (vibeisland.app) is a paid native macOS app (Swift/AppKit, no Electron, bundle id app.vibeisland.macos) by solo developer Edward Luo (@imedwardluo, GitHub edwluo) that turns the MacBook notch into a "Dynamic Island" control surface for 26 AI coding agents (Claude Code, Codex CLI+Desktop, Gemini CLI, Cursor, OpenCode, Copilot CLI, Amp, Droid, Kiro, Qwen, Kimi, DeepSeek, Grok Build, etc.). It shows live session status in a compact notch pill (or a floating top-center bar on non-notch/external displays) and expands into interactive cards to approve/deny tool permissions (Allow ⌘Y / Deny ⌘N with code diffs), answer agent questions (multiple-choice or free text), review plan-mode plans with full Markdown rendering, and click-to-jump to the exact terminal tab/split pane (20+ terminals via AppleScript/Apple Events, plus VS Code/Cursor via a bundled "terminal-focus" IDE extension using a URI handler + terminal processId matching). Technically it is far more than hooks: a resident app listens on Unix socket /tmp/vibe-island.sock; a bundled helper (~/.vibe-island/bin/vibe-island-bridge wrapper → Contents/Helpers/vibe-island-bridge) is auto-installed into ~/.claude/settings.json (11 Claude Code hook events incl. a blocking PermissionRequest hook with timeout 86400 that enables GUI approval), ~/.codex/hooks.json, ~/.gemini/settings.json, ~/.cursor/hooks.json; Claude usage quota comes from a statusLine bridge (~/.vibe-island/bin/vibe-island-statusline writes rate_limits to /tmp/vibe-island-rl.json) plus api.anthropic.com/api/oauth/usage; Codex additionally gets rollout-JSONL watching under ~/.codex/sessions and a Codex app-server JSON-RPC/WebSocket client with a CodexDesktopApprovalWatcher; OpenCode gets a JS plugin in ~/.config/opencode/plugins/vibe-island.js that replies to permissions/questions over localhost HTTP. Session state persists in ~/Library/Application Support/vibe-island/session-terminals.json. Pricing is $19.99 one-time (1/2/3-Mac tiers, 1 year updates, perpetual use afterward, 2-day free trial, no card), macOS 14 Sonoma+, Apple Silicon, <50-100MB RAM, installed via DMG or `brew install --cask vibe-island` (v1.0.42, Jul 18 2026). Reception is strong especially in the Chinese dev community (maker's X thread, hi_caicai endorsement, Idea Browser calling it "the best mac app of 2026" and giving away 10 licenses); it launched as v0.5.0 beta on Mar 3 2026 and spawned multiple clones (open-vibe-island 629 stars in 17 days, X Island on Product Hunt, MioIsland, vibe-notch, Notchi, AgentNotch); no official Product Hunt or Hacker News launch was found, and vibeisland.lol is an unrelated placeholder/copycat site. Notable caveat: marketing claims "no telemetry" but the binary contains Sentry and PostHog endpoints plus LemonSqueezy licensing (per the open-vibe-island bundle analysis).

## Key findings
- WHAT IT IS: macOS notch panel ('Dynamic Island') for AI coding agents: live session status, GUI permission approval, question answering, plan review, usage-limit tracking, and precise jump-back to the originating terminal tab/pane or IDE window. Native Swift, no Electron, <50-100MB RAM, near-zero idle CPU. Developer: Edward Luo (solo), GitHub org vibeislandapp, updates repo edwluo/vibe-island-updates.
- SUPPORTED AGENTS (26): Claude Code, Codex (CLI + Codex Desktop app), ZCode, Gemini CLI, Antigravity CLI, Cursor, Trae, OpenCode, MiMoCode, Droid, Qoder, Qwen, Grok Build, Kimi Code, Kimi, DeepSeek, Mistral Vibe, Copilot, CodeBuddy, WorkBuddy, Kiro, Hermes, Amp, Pi Agent, Oh My Pi, Gajae Code. Also VS Code native extension path and SSH Remote monitoring of agents on remote hosts (hooks deployed over SSH tunnels, isolated per-host ports).
- CLAUDE CODE INTEGRATION (confirmed from real user dotfiles + GitHub issues): auto-injects hooks into ~/.claude/settings.json for Notification, PermissionRequest (timeout 86400 = blocking 24h so the notch UI can answer synchronously), PreToolUse, PostToolUse, PreCompact, SessionStart, SessionEnd, SubagentStart, SubagentStop, Stop, UserPromptSubmit — all running `$HOME/.vibe-island/bin/vibe-island-bridge --source claude` (wrapper that execs Contents/Helpers/vibe-island-bridge inside the app bundle; app path cached in ~/.vibe-island/bin/.bridge-cache). Events flow over Unix socket /tmp/vibe-island.sock to the resident app.
- CLAUDE USAGE TRACKING: installs statusLine.command = ~/.vibe-island/bin/vibe-island-statusline in ~/.claude/settings.json; the script reads Claude Code statusline JSON from stdin and writes the .rate_limits payload to /tmp/vibe-island-rl.json; binary also references https://api.anthropic.com/api/oauth/usage. Compact notch shows quota glance like '5h 11% | 7d 2%' (5-hour and 7-day windows) for Claude, Codex, Kimi, GLM, DeepSeek. It refuses to overwrite an existing custom statusLine (asks user to fix/remove it first).
- CODEX INTEGRATION: ~/.codex/hooks.json wires SessionStart, UserPromptSubmit, Stop to `vibe-island-bridge --source codex`; sets [features] codex_hooks = true in ~/.codex/config.toml; watches rollout JSONL session files under ~/.codex/sessions and ~/.codex/session_index.jsonl (session-terminals.json stores codexRolloutPath/codexOrigin/codexThreadSource, tracks subagents); binary contains CodexAppServerClient/CodexAppServerManager/CodexDesktopApprovalWatcher — spawns `/Applications/Codex.app/Contents/Resources/codex app-server` and speaks JSON-RPC/WebSocket (thread/status/changed, waitingOnApproval) for Codex Desktop approvals.
- OTHER AGENTS: Gemini CLI — hooks in ~/.gemini/settings.json (BeforeAgent, AfterAgent, BeforeTool, AfterTool, SessionStart, SessionEnd). Cursor — ~/.cursor/hooks.json (beforeSubmitPrompt, beforeShellExecution, afterShellExecution, beforeMCPExecution, afterMCPExecution, plus response/file/stop hooks). OpenCode — auto-installed JS plugin ~/.config/opencode/plugins/vibe-island.js: connects to /tmp/vibe-island.sock, derives TTY by walking the process tree, sets Ghostty OSC 2 tab titles, and answers questions/permissions via http://localhost:{port}/question/{id}/reply and /permission/{id}/reply; the app also connects to the native OpenCode server at http://127.0.0.1:4096.
- TERMINAL JUMP MECHANISM: Apple Events/AppleScript automation (entitlement com.apple.security.automation.apple-events; 'tell application iTerm2/Terminal' strings in binary) plus per-terminal adapters (cmux socket/surface/workspace IDs persisted per session); for VS Code/Cursor it installs extension vibe-island.terminal-focus-1.0.0 into ~/.vscode/extensions and ~/.cursor/extensions which registers a URI handler, parses a pid query param, and focuses the integrated terminal whose processId matches. Supported terminals: iTerm2, Terminal.app, Ghostty, Warp, WezTerm, Kitty, Alacritty, Hyper, Zed, Zellij, cmux, Conductor, Termius, Kaku, Otty, Supacode, Superset, herdr, tmux — including split panes and tabs.
- SESSION STATE: ~/Library/Application Support/vibe-island/session-terminals.json persists per-session cwd, tty, currentTool, status, source, firstUserMessage, lastUserMessage, lastAssistantMessage(Full), codex* fields. Hook install is a managed subsystem (settings.json.backup/hooks.json.backup, hookRepairTimestamps, rate-limited auto-repair com.vibe-island.repairHooks, symlink-safe config writer, JSONC comment parsing). Zap paths: ~/.vibe-island, ~/Library/Caches/app.vibeisland.macos, ~/Library/Preferences/app.vibeisland.macos.plist.
- UI/UX FLOWS: Compact state = pill in/around the notch (floating top-center bar on non-notch Macs/external displays) with session cards (agent icon, project name, git branch, optional model name, status: working/waiting/needs approval/done) and usage quota glance. Expanded state = triggered when input is needed: permission card shows e.g. '⚠ Edit src/auth/middleware.ts' with code diff and 'Deny ⌘N / Allow ⌘Y' buttons (covers file edits, shell commands, MCP calls; 'Always allow' option exists); question card shows the question with one-click options or free-text input; plan-mode card renders the full plan as Markdown (headings, code blocks, lists) with approve-or-type-feedback to send the agent back to revise. Cards queue when multiple approvals are pending. Clicking a session jumps to its exact terminal pane.
- NOTIFICATIONS/SOUND: macOS notification on task completion (click = jump to that terminal); '8-bit synthesized sounds for every event' plus custom sound packs (pack registry hosted at PeonPing.github.io/registry/index.json); 'Quiet scenes' auto-silence during macOS Focus, screen lock, or screen sharing; per-approval-card modes 'Follow focus / Notify me / Stay silent'; 'Glance completion mode' keeps the panel collapsed with a subtle indicator instead of auto-expanding; subagent display controls for fan-out panels.
- PRICING & REQUIREMENTS: $19.99 one-time purchase (tiers for 1, 2, or 3 Macs; AlternativeTo still lists the older $15 price), includes 26-agent support, GUI approval and question answering, terminal jump, unlimited sessions, and 1 year of updates; app keeps working with all features after the year, renewal optional; 2-day free trial, no card; licensing via LemonSqueezy; license transferable to a new Mac. Requirements: macOS 14 Sonoma+ (Homebrew cask depends_on :sonoma), Apple Silicon-optimized. Install: https://dl.vibeisland.app/VibeIsland.dmg or `brew install --cask vibe-island`; current version 1.0.42 (Jul 18, 2026); Sparkle auto-updates via updates.vibeisland.app/appcast.xml and edwluo.github.io/vibe-island-updates/appcast.xml.
- PRIVACY CLAIM VS REALITY: marketing says 'No cloud, no accounts, no telemetry'; the open-vibe-island bundle analysis (v1.0.15) found Sentry crash-reporting and PostHog analytics (us.i.posthog.com/capture) endpoints in the binary plus LemonSqueezy license checks — agent/session data does stay local, but the no-telemetry claim is contradicted at binary level.
- RECEPTION: launched v0.5.0 beta Mar 3, 2026; huge traction in the Chinese dev community (maker's launch thread in Chinese on X @imedwardluo describing running 5-10 parallel vibe-coding sessions and losing track; endorsement by @hi_caicai; Idea Browser @ideabrowser: 'This is the best mac app of 2026... giving away 10 licenses'). Spawned clones: Octane0411/open-vibe-island (open-source, 629 stars in 17 days; hooks→Unix-socket bridge architecture mirroring the original), 'X Island' on Product Hunt (114 upvotes, maker @bluedusk — separate clone product), MioMioOS/MioIsland, farouqaldori/vibe-notch (ex 'Claude Island'), Notchi, AgentNotch, Open Island. No official Product Hunt launch or Hacker News thread found for Vibe Island itself. vibeisland.lol is an unrelated placeholder/copycat landing page.
- KNOWN FRICTION (GitHub issues, vibeislandapp/vibe-island — issues-only community repo, 170+ issues): Stop hook blocks Claude Code shutdown ~15-21s (open #159); OpenCode sessions duplicated when both Claude and OpenCode auto-config enabled (#155); sessions in .claude/worktrees/ previously invisible due to path encoding (#149/#150, fixed); Codex Desktop subagents stuck 'processing' (#143, fixed); notch intercepts clicks during Claude computer-use (#171); no multi-display panels yet (#168); SSH tunnel reconnect loop can lock up the machine (#160). Support: Discord discord.gg/2GmjPVtc3d, hi@vibeisland.app, changelog at vibeisland.app/changelog/.

## Recommendations
- If the goal is to build a similar app: replicating it requires six layers, not just hooks — (1) hook ingress per CLI, (2) a resident Unix-socket bridge server, (3) per-tool adapters beyond hooks (Codex app-server JSON-RPC, OpenCode JS plugin, statusline bridge for quota), (4) IDE extensions with URI handlers for VS Code/Cursor terminal focus, (5) Apple Events/AppleScript terminal automation (needs com.apple.security.automation.apple-events entitlement and user consent), (6) a persisted session index (session-terminals.json equivalent) for jump-back and reconciliation.
- Use Octane0411/open-vibe-island as the reference implementation — its docs/installed-app-bundle-analysis.md is a full reverse-engineering of Vibe Island v1.0.15 and its own architecture (hook event → CLI helper → stdin → Unix socket → JSON envelope → in-app BridgeServer) mirrors the commercial app; MioMioOS/MioIsland has a readable CodexHookInstaller.swift.
- For Claude Code specifically, the two key tricks are: a PermissionRequest hook with a very long timeout (86400s) whose stdout JSON decision is deferred until the user clicks Allow/Deny in the GUI, and a statusLine command wrapper to harvest rate_limits JSON for quota display — both are injected into ~/.claude/settings.json with backup + auto-repair logic and symlink-safe writes.
- Avoid Vibe Island's known pitfalls if cloning: make the Stop hook fire-and-forget (issue #159 shows a blocking teardown adds 15-21s to Claude Code exit), deduplicate sessions when one runtime (OpenCode) also triggers another CLI's hooks (#155), normalize worktree paths under .claude/worktrees (#149), and don't intercept mouse events over the notch region when agents drive the UI (#171).
- For further primary-source detail, fetch the follow-up doc app-ghostty-codex-chain.md in open-vibe-island/docs, the OpenCode plugin file ~/.config/opencode/plugins/vibe-island.js from an installed copy, and the release feed at github.com/edwluo/vibe-island-updates/releases for per-version UX changes.
