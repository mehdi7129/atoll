# session-liveness

## Summary
Session liveness for a macOS Claude Code monitor is solved by combining three signal layers, all verified empirically on Claude Code 2.1.214 and cross-checked against three open-source implementations. (1) PID capture: hook payloads contain session_id/transcript_path/cwd but NO pid; the hook process's getppid() IS the claude PID (Claude spawns the hook command as a direct child shell which execs simple commands), so every hook forwards {session_id, pid=getppid(), cwd, event} to the app — vibe-notch does exactly this over a Unix socket at /tmp/claude-island.sock. (2) Exit watching: DispatchSource.makeProcessSource(identifier:pid, eventMask:.exit) works on non-child, unrelated same-user processes (proven with a compiled Swift test watching a launchd-reparented process); the only race is a PID that dies before arming, closed by a post-resume kill(pid,0)==ESRCH check; PID identity across app restarts is guaranteed by pairing pid with proc start time from proc_pidinfo(PROC_PIDTBSDINFO).pbi_start_tvsec. (3) Fallback discovery: enumerate processes (ps -Ao or proc_listpids+proc_name, since proc_pidpath resolves to ~/.local/share/claude/versions/2.1.214 whose basename is NOT "claude"), read cwd via lsof -d cwd / PROC_PIDVNODEPATHINFO, and map to the newest-mtime .jsonl under ~/.claude/projects/<munged-cwd>/ whose internal per-line "cwd" field matches — lsof on open transcript fds is UNRELIABLE because claude 2.1.214 holds zero .jsonl fds open (append-and-close). SessionEnd DOES fire on SIGHUP (terminal closed/SSH drop) and SIGTERM with reason "other" (tested), but NOT on SIGKILL, hard crash, or kills during the startup/trust screen — those are the only true ghost sources, which the kqueue exit watch plus a 2-consecutive-miss process-scan debounce (open-vibe-island's pattern) eliminates. State discrimination: busy = UserPromptSubmit/PreToolUse/PostToolUse stream; waiting-on-permission = PermissionRequest hook (blockable, can even remote-approve) or Notification matcher permission_prompt; waiting-for-input = Stop hook or Notification idle_prompt; dead = kqueue NOTE_EXIT or SessionEnd hook.

## Key findings
- Hook payload fields (verified 2.1.214): all hooks receive JSON on stdin with session_id, transcript_path, cwd, hook_event_name (+prompt_id after first input); there is NO pid field — the hook script must capture it itself.
- getppid() chain (verified empirically): Claude spawns hook commands as /bin/zsh -c '<command>' which is a DIRECT child of the claude process, and the shell execs simple commands — so os.getppid() in the hook == claude PID. For compound commands, walk up ppid via proc_pidinfo(PROC_PIDTBSDINFO).pbi_ppid until proc_name(pid)=='claude'. vibe-notch's claude-island-state.py uses plain os.getppid() and it works.
- Forwarding transports used in the wild: vibe-notch = Unix domain socket /tmp/claude-island.sock (SOCK_STREAM, JSON blob per connection; PermissionRequest connections stay open up to 300s waiting for an allow/deny decision from the app); gmr/claude-status = per-session ~/.claude/projects/<proj>/<session_id>.cstatus JSON file {session_id,pid,ppid,state,activity,timestamp,cwd,event} plus Darwin notification 'notifyutil -p com.poisonpenllc.Claude-Status.session-changed' consumed via CFNotificationCenterGetDarwinNotifyCenter; open-vibe-island = compiled hooks binary + local bridge server.
- DispatchSource.makeProcessSource(identifier: pid_t, eventMask: .exit, queue:) — VERIFIED working on a non-child, non-descendant process (watched a launchd-reparented sleep; event fired at exact exit). Underlying kqueue EVFILT_PROC/NOTE_EXIT; man 2 kqueue: 'If a process can normally see another process, it can attach an event to it' — same-user watching needs no privileges, no root, no entitlements (App Sandbox would break it; ship unsandboxed/Developer ID).
- Arming race (verified): if the PID is already dead when the source is created, the event handler NEVER fires and no error surfaces. Mandatory pattern: create source → resume() → if kill(pid,0) == -1 && errno == ESRCH, treat as already-dead. proc_pidinfo also fails with errno 3 (ESRCH) for dead pids.
- PID-reuse guard: once armed, the kqueue knote is attached to the specific proc instance, so reuse cannot fool an armed watch; the hazard is only at (re)validation time. Robust identity = (pid, start_time) tuple where start_time = proc_bsdinfo.pbi_start_tvsec/pbi_start_tvusec via proc_pidinfo(pid, PROC_PIDTBSDINFO, 0, &info, size). Store it when the first hook arrives; on app restart, a persisted session is live only if pid exists AND start time matches. gmr/claude-status instead re-checks proc_pidpath contains 'claude'/'Claude' or ends '/node' — weaker but adequate.
- Executable-name trap (verified locally): the native installer's binary is ~/.local/bin/claude but proc_pidpath returns the REAL executable ~/.local/share/claude/versions/2.1.214 (basename '2.1.214', lsof COMMAND column shows '2.1.214'); proc_name/ps comm returns 'claude'. Match with proc_name == 'claude' OR proc_pidpath containing '/claude' — never by basename == 'claude' alone. Also filter out false positives like 'Cursor Helper ... claude' argv matches by requiring first argv token/basename, and skip subagent worktree processes (cwd under .claude/worktrees/agent-*).
- Sleep/wake: kqueue registrations persist across system sleep; a process cannot exit while the kernel sleeps, so NOTE_EXIT cannot be missed — at worst delivery is deferred to wake. App Nap can delay the GCD handler; take NSProcessInfo.processInfo.beginActivity(.background) or rely on the menu-bar app's UI activity. Registrations do NOT survive the watcher's own relaunch — revalidate all persisted sessions with (pid,starttime) on startup.
- Transcript ground truth: ~/.claude/projects/<munged-cwd>/<session-uuid>.jsonl where munging replaces every non-alphanumeric char (/, ., _, space) with '-' (verified: /Users/mehdiguiard/Desktop/Dynamic_Island → -Users-mehdiguiard-Desktop-Dynamic-Island); mapping is lossy/collision-prone, so confirm by parsing the 'cwd' and 'sessionId' fields present on every JSONL line. Process cwd for the match comes from lsof -a -d cwd -p PID or proc_pidinfo(PROC_PIDVNODEPATHINFO).pvi_cdir.vip_path.
- lsof-open-transcript detection is DEAD on 2.1.214: the claude process holds 0 open .jsonl fds (52 total fds, none under ~/.claude) — it opens/appends/closes per write. open-vibe-island's ActiveAgentProcessDiscovery still greps 'lsof -p' output for '/.claude/projects/…​.jsonl' paths (works only opportunistically mid-write) and falls back to session ID from argv (--resume <uuid>) and TTY+CWD matching; do not build on lsof fd presence.
- SessionEnd firing matrix (empirically tested on 2.1.214, interactive session under a pty): SIGHUP (= terminal window closed, SSH disconnect) → SessionEnd fires, reason 'other'; SIGTERM → fires, reason 'other'; SIGKILL → does NOT fire; claude -p completion → fires, reason 'other'; kill during startup/folder-trust screen (before SessionStart) → nothing fires. Documented reason values: clear, resume (session resumed elsewhere), logout, prompt_input_exit, bypass_permissions_disabled, other. System sleep does not end sessions at all.
- Known SessionEnd reliability bugs (anthropics/claude-code issues): #17885/#35892 SessionEnd not fired on /exit; #32712 SessionEnd hook cancelled with 'Request interrupted by user' when exiting via Ctrl+C; #41577 SessionEnd hooks killed before async work completes (even with 90s timeout — keep the hook to a fast socket write); #33458 plugin-defined SessionEnd hooks never fire (define in user settings.json, not a plugin). Treat SessionEnd as best-effort, never as the sole death signal.
- Hook events available for state discrimination (2.1.x): SessionStart (matchers startup|resume|clear|compact; 2.1.207 adds source 'fork'), UserPromptSubmit, PreToolUse, PostToolUse, PostToolUseFailure, PermissionRequest (blockable — can allow/deny via hookSpecificOutput.decision.behavior), PermissionDenied, Notification (matchers permission_prompt | idle_prompt | agent_needs_input | agent_completed | elicitation_*), Stop (fires when Claude finishes responding), StopFailure (matchers rate_limit|overloaded|authentication_failed|billing_error|server_error|…), SubagentStart/SubagentStop, PreCompact/PostCompact, SessionEnd. Notification/idle_prompt fires when Claude has been sitting at the prompt awaiting input (~60s idle).
- Event→state mapping used by vibe-notch (claude-island-state.py): UserPromptSubmit→processing; PreToolUse→running_tool; PostToolUse/PostToolUseFailure/PermissionDenied/SubagentStart/SubagentStop→processing; PermissionRequest→waiting_for_approval (blocks for app decision); Notification[idle_prompt]→waiting_for_input (Notification[permission_prompt] deliberately skipped in favor of PermissionRequest); Stop/StopFailure→waiting_for_input; SessionStart→waiting_for_input; PreCompact→compacting; PostCompact→processing; SessionEnd→ended.
- vibe-notch expiry: 3-second periodic task; removes a session when its phase is .ended or kill(pid,0)!=0 (no PID-reuse guard, no debounce). Its SessionPhase enum {idle, processing, waitingForInput, waitingForApproval(PermissionContext), compacting, ended} has an explicit canTransition() validation table; .ended is terminal; needsAttention = waitingForApproval|waitingForInput.
- vibe-notch interrupt detection (Esc pressed mid-turn — no hook exists for this): a DispatchSource.makeFileSystemObjectSource([.write,.extend]) tail-watcher on the session .jsonl fires on new lines matching '[Request interrupted by user]', tool_result with is_error:true + 'Interrupted by user', or '"interrupted":true' → flips processing→waitingForInput instantly.
- gmr/claude-status expiry: scans .cstatus files, kill(pid,0) + proc_pidpath('claude'|'node') reuse-guard; confirmed-dead session IDs cached in a deadSessions set to skip rescans until a Darwin notification clears it. Refresh triggers: Darwin notification (instant push), DispatchSource file watch on ~/.claude/projects, 5s polling timer.
- gmr/claude-status hookless JSONL fallback heuristics (StateResolver): mtime <5s → active; else parse last user/assistant line from a 64KB tail: last line assistantWorking (stop_reason != end_turn, or user line containing tool_result) → active if <30s else waiting; assistantDone (stop_reason == end_turn) → active if <10s else waiting; plain user message → active if <30s else waiting; nothing meaningful → idle.
- open-vibe-island expiry (SessionState.markProcessLiveness): hook-managed sessions are trusted until SessionEnd, BUT as a bridge-loss fallback each full reconcile matches sessions to live claude processes in 3 passes (1: exact sessionID from process, 2: transcriptPath match, 3: TTY+CWD match); a session missed by 2 CONSECUTIVE reconcile passes (processNotSeenCount >= 2) is marked ended/completed and removed — this debounce prevents a single failed ps scan from killing live sessions. Reconcile cadence: 2s during startup resolution, 60s active, 300s idle; hookless 'synthetic' sessions are created for discovered claude processes with no hook events; desktop-app-tracked rows use a 600s staleness timeout after .completed.
- open-vibe-island process enumeration: single '/bin/ps -Ao pid=,ppid=,tty=,command=' pass (0.5s timeout) instead of libproc; claude processes must be TTY-attached; terminal app identified by walking ppid chain to a recognized *.app path; tmux resolved separately because the tmux server reparents to launchd (pid 1) breaking the ancestor walk — detect via TMUX/TMUX_PANE env read from KERN_PROCARGS2 sysctl (gmr's readEnvironmentVariable shows the exact KERN_ARGMAX/KERN_PROCARGS2 parsing code).
- libproc alternative to ps: proc_listpids(PROC_ALL_PIDS, 0, buf, size) → proc_name(pid)/proc_pidpath(pid) filter → proc_pidinfo(PROC_PIDTBSDINFO) for ppid+start-time → proc_pidinfo(PROC_PIDVNODEPATHINFO) for cwd. All work unprivileged for same-user processes (import Darwin, no private headers; declare via libproc.h).
- Hook health checking (open-vibe-island HookHealthCheck): detect broken installs — hooks binary missing/not executable, settings.json malformed, stale command path, plugin-defined hooks (which silently don't fire for SessionEnd per #33458) — and surface repair; hook loss is a first-class failure mode, not an edge case.
- Waiting-on-permission detection options ranked: (1) PermissionRequest hook — richest (tool_name, tool_input, tool_use_id), blockable so the app can approve/deny remotely (vibe-notch responds over the still-open socket with hookSpecificOutput.decision.behavior allow/deny); note default hook timeout 600s auto-falls-through; (2) Notification hook matcher permission_prompt — fire-and-forget signal only; (3) transcript tail stuck on an assistant tool_use with no following tool_result while no PostToolUse arrives — weakest. Clear pending permission state on Stop, PostToolUse (same tool_use_id), or interrupt detection.
- SessionStart(source=clear) + SessionEnd(reason=clear) fire on /clear with a NEW session_id continuing in the same process/PID — same PID, different session; key by session_id but bind PID at SessionStart, and on SessionEnd(reason=resume/clear) expect a successor session on the same PID rather than marking the terminal dead.

## Recommendations
- Recommended state machine — states: STARTING (SessionStart received), BUSY (turn in progress), TOOL_RUNNING (optional sub-state with tool name), WAITING_PERMISSION(toolUseId, toolName, input), COMPACTING, WAITING_INPUT (at prompt), DEAD. Transitions driven by hook events exactly as vibe-notch maps them (UserPromptSubmit→BUSY; PreToolUse→TOOL_RUNNING; PostToolUse/PostToolUseFailure/PermissionDenied/SubagentStop→BUSY; PermissionRequest→WAITING_PERMISSION; Stop/StopFailure/Notification[idle_prompt]→WAITING_INPUT; PreCompact→COMPACTING; PostCompact→BUSY; SessionEnd→DEAD). Validate transitions with an explicit canTransition table (copy vibe-notch SessionPhase.swift); DEAD is terminal per session_id.
- PID plumbing: install hooks for SessionStart, UserPromptSubmit, PreToolUse, PostToolUse, PostToolUseFailure, PermissionRequest, Notification, Stop, StopFailure, PreCompact, PostCompact, SubagentStop, SessionEnd in ~/.claude/settings.json (never in a plugin — SessionEnd won't fire). The hook script: read stdin JSON, add pid=getppid() (walk up until proc name == 'claude' for robustness), tty from 'ps -p $ppid -o tty=', write one JSON message to the app's Unix socket with a short (<1s) connect timeout and exit 0 on ANY failure so hooks never break the CLI.
- Liveness core: on first event for a session_id, record (pid, start_time via proc_pidinfo PROC_PIDTBSDINFO) and arm DispatchSource.makeProcessSource(identifier: pid, eventMask: .exit); after resume(), check kill(pid,0)==-1&&errno==ESRCH → immediately DEAD. On the exit event: mark DEAD instantly (this is what kills ghosts from SIGKILL/crash with zero polling). One source per unique PID; sessions sharing a PID (after /clear) share the watch.
- Keep a slow reconcile safety net (open-vibe-island pattern) for hook-loss: every 30–60s enumerate claude processes (ps -Ao pid,ppid,tty,command or proc_listpids; match proc_name=='claude' or path containing '/claude'), map to sessions by pid → argv --resume uuid → cwd+newest-mtime transcript (verify inner cwd field); a tracked session unmatched on 2 CONSECUTIVE passes → DEAD (debounce); an unmatched claude process → create a synthetic session in WAITING_INPUT resolved via transcript tail heuristics (mtime <5s → BUSY; last assistant stop_reason==end_turn → WAITING_INPUT; trailing tool_use without tool_result → likely WAITING_PERMISSION or BUSY).
- Expiry rules: never time-expire a session whose PID watch is armed and silent — trust the kernel. Time-based GC only for sessions with no known PID (hookless/synthetic): expire after 10 min without transcript mtime change (open-vibe-island uses 600s). On app launch, restore persored sessions only if (pid,start_time) still matches, else drop silently. Treat SessionEnd(reason=clear|resume) as session-swap (expect successor on same PID within ~5s) rather than terminal death of the terminal tab.
- Freshness beats polling: use DispatchSource.makeFileSystemObjectSource on ~/.claude/projects (O_EVTONLY) for new-session discovery, and a per-session .jsonl tail watcher while BUSY to catch Esc-interrupts (match '[Request interrupted by user]' / is_error:true patterns) — this is the only way to catch user interrupts since no hook fires for them.
- For WAITING_PERMISSION prefer the blockable PermissionRequest hook and keep the socket open for remote approve/deny from the notch UI (vibe-notch's flow, decision JSON: hookSpecificOutput.decision.behavior allow|deny); set an explicit hook timeout and auto-fall-through to Claude's own prompt on app absence.
- Ship the app unsandboxed (Developer ID + notarization) — kqueue EVFILT_PROC on other processes, KERN_PROCARGS2 env reads, and ~/.claude access are all incompatible with App Sandbox.
