# ascii-ui

## Summary
Research complete on building an ASCII/terminal-retro aesthetic in native SwiftUI on macOS. Fonts: SF Mono cannot be bundled (Apple Font License restricts it to Apple-branded apps/mockups) but is legally usable at runtime as the system monospaced font via Font.system(design: .monospaced) / NSFont.monospacedSystemFont; JetBrains Mono, IBM Plex Mono, and Departure Mono are all SIL OFL 1.1 and safe to bundle (via ATSApplicationFontsPath in Info.plist on macOS); Berkeley Mono (TX-02, usgraphics.com) is commercial and explicitly disallows app bundling under standard licenses. Characters: box-drawing U+2500-257F, block elements U+2580-259F (partial blocks give 1/8-cell progress-bar resolution), braille U+2800-28FF with the classic cli-spinners "dots" frames at 80ms. Effects: macOS 14+ SwiftUI Metal shaders (.colorEffect/.distortionEffect/.layerEffect with [[stitchable]] functions) are the right tool for scanlines/CRT/noise — twostraws/Inferno (MIT) ships ready-made Interlace, White Noise, and Shimmer shaders; glow is best done with stacked .shadow() calls or a blurred duplicate underlay. Theming: @Environment(\.colorScheme) to read, .preferredColorScheme per-window, NSApp.appearance = NSAppearance(named: .darkAqua/.aqua) for app-wide override and nil to follow the system, persisted via @AppStorage. Animation: TimelineView(.periodic(by: 0.08)) driving a frame index into a static frames array is the idiomatic spinner/ASCII-animation pattern; a single multi-line monospaced Text (or AttributedString for color spans) is fast enough for ~80x24 grids at 12.5fps, while Canvas + context.resolve(Text) is needed for 60fps full-surface effects. Design references: terminal.shop (stark B&W + orange accent), sacred.computer/SRCL (precise monospace spacing, MS-DOS cards), Berkeley Graphics/usgraphics.com (engineering-datasheet density), and cool-retro-term (portable CRT shader reference).

## Key findings
- SF Mono licensing: Apple Font License permits use 'solely in conjunction with Apple-branded applications' and for UI mockups of apps on Apple platforms — you may NOT bundle SF Mono font files in a shipped third-party app (developer.apple.com/fonts). The compliant route: SF Mono ships with macOS as the system monospaced font, so use runtime APIs — SwiftUI `Font.system(size:weight:design:)` with `.monospaced`, `.fontDesign(.monospaced)`, `Text.monospaced()`, `.monospacedDigit()`, or AppKit `NSFont.monospacedSystemFont(ofSize:weight:)` — no font file is embedded, so no license issue.
- Bundleable OFL fonts: JetBrains Mono (github.com/JetBrains/JetBrainsMono, SIL OFL 1.1, weights 100-800 + italics, 148 ligatures — consider disabling ligatures for grid-true ASCII UI), IBM Plex Mono (github.com/IBM/plex, OFL 1.1, on Google Fonts), Departure Mono (departuremono.com, by Helena Zhang, OFL, lo-fi pixel aesthetic, 775 glyphs including simple box-drawing; render at integer multiples of its native pixel size for crispness; npm/packaging mirror at github.com/rektdeckard/departure-mono). SIL OFL 1.1 explicitly allows bundling/embedding in commercial apps as long as fonts aren't sold standalone.
- Berkeley Mono (TX-02, usgraphics.com/products/berkeley-mono, Berkeley Graphics / U.S. Graphics Company): commercial only. Developer license (FX-102) covers personal/device install, not redistribution; app bundling in IDEs/terminals/editors is generally NOT allowed without contacting them; commercial licenses are incompatible with open-source apps. Use it as an aesthetic reference, not a bundled asset.
- Bundling custom fonts in a macOS app: add .ttf/.otf files to a folder (e.g. 'Fonts') in Resources and set Info.plist key `ATSApplicationFontsPath = Fonts` (macOS equivalent of iOS `UIAppFonts`); alternative runtime registration via `CTFontManagerRegisterFontsForURL(url, .process, nil)`. Then `Font.custom("JetBrainsMono-Regular", size: 13)`.
- Character sets: box-drawing U+2500-257F (light ─│┌┐└┘├┤┬┴┼, heavy ━┃┏┓┗┛, double ═║╔╗╚╝, rounded corners ╭╮╰╯); block elements U+2580-259F — shade blocks ░▒▓█ and eighth-blocks ▏▎▍▌▋▊▉█ give 8x sub-character resolution for smooth ASCII progress bars (index = Int(fraction * cellCount * 8) % 8 for the partial cell); vertical eighths ▁▂▃▄▅▆▇█ for sparklines; braille patterns U+2800-28FF for spinners and 2x4-dot pseudo-pixel graphics.
- Spinner frames: sindresorhus/cli-spinners (MIT, github.com/sindresorhus/cli-spinners) is the canonical source — spinners.json holds ~80 spinners as {interval_ms, frames[]}, trivially portable to a Swift enum. The classic 'dots' braille spinner is frames ⠋⠙⠹⠸⠼⠴⠦⠧⠇⠏ at 80ms; keep all frames within one Unicode block so glyph widths match.
- Animating ASCII frames: `TimelineView(.periodic(from: .now, by: 0.08)) { ctx in let i = Int(ctx.date.timeIntervalSinceReferenceDate / 0.08) % frames.count; Text(frames[i]) }` — no Timer, no @State churn; use `.animation(minimumInterval:paused:)` schedule only for continuous shader time uniforms. Wrap the swapped Text in `.contentTransition(.identity)` (or `.numericText()` for counters) and an explicit `Transaction(animation: nil)` to prevent implicit crossfades.
- Text vs Canvas performance: a single multi-line `Text` with a monospaced font, `.lineSpacing(0)`, `.kerning(0)` is one CoreText layout pass and comfortably handles an 80x24 (1920-char) grid at 10-15fps updates; use `AttributedString` runs for multi-color spans inside one Text instead of per-character Text views (never build HStack/VStack grids of single-character Texts — view-tree explosion). For 60fps full-surface effects (matrix rain, plasma), use `Canvas` inside TimelineView: `let t = context.resolve(Text("█").font(mono)); context.draw(t, at: point)` — resolve once per glyph/style, draw many times. `.drawingGroup()` flattens static ASCII compositions into one Metal layer before applying blur/shadow effects. Box-drawing characters only connect vertically if line height equals the glyph cell height — set `.lineSpacing(0)` and test the chosen font (SF Mono and JetBrains Mono box glyphs connect correctly).
- CRT/scanline/glow effects (macOS 14+): SwiftUI native Metal shaders via `.colorEffect(ShaderLibrary.myShader(...))` (per-pixel color, e.g. scanlines: dim every other 2px row via `fmod(position.y, 4.0) < 2.0`), `.distortionEffect` (barrel/CRT curvature), `.layerEffect` (bloom/glow via neighbor sampling); shader signature `[[stitchable]] half4 name(float2 position, half4 color, float time)`. Ready-made MIT library: github.com/twostraws/Inferno (iOS 17/macOS 14, Swift 5.9, MSL 3.1) includes Interlace (exact scanline effect: alternating horizontal lines of a second color with adjustable width/strength), White Noise, Rainbow Noise, Shimmer, Emboss, Light Grid, plus 13 transitions (Pixellate, Swirl, Genie). Cheaper non-Metal glow: stacked `.shadow(color: accent.opacity(0.8), radius: 1-2)` + `.shadow(color: accent.opacity(0.4), radius: 6-10)` for phosphor bloom, or a `.background(sameText.blur(radius: 6).opacity(0.5))` duplicate; apply once to a container (with drawingGroup), never per-glyph. Vignette = radial gradient overlay; flicker = TimelineView-driven opacity jitter in 0.97-1.0 range. Reference CRT shader implementation to port: github.com/Swordfish90/cool-retro-term (GLSL/QML).
- macOS theming: read with `@Environment(\.colorScheme)`; `.preferredColorScheme(.dark/.light)` overrides per-window/scene (pass nil to defer); for a true app-wide override including title bars, menus, and panels set `NSApp.appearance = NSAppearance(named: .darkAqua)` / `NSAppearance(named: .aqua)`, and `NSApp.appearance = nil` to follow the system (auto mode). Standard pattern: `enum Theme: String { case system, light, dark }` in `@AppStorage("theme")`, applied in App init and `.onChange`. Observe external system flips via KVO on `NSApp.effectiveAppearance` or DistributedNotificationCenter 'AppleInterfaceThemeChangedNotification'. Define all palette colors as Assets.xcassets color sets with Any/Dark variants (or a ShapeStyle-returning theme struct keyed off colorScheme) so auto theming is free; check contrast via `@Environment(\.colorSchemeContrast)`.
- Terminal color palettes (hex): phosphor green #33FF33 or #00FF41 on near-black #0D0208; amber phosphor #FFB000 on #1A1000; Solarized (dark bg #002B36/fg #839496, light bg #FDF6E3 — built for paired light/dark, ideal for auto theming); Gruvbox (dark #282828/#EBDBB2, light #FBF1C7); Nord (#2E3440 bg, #D8DEE9 fg, #88C0D0 frost accent); Dracula (#282A36/#F8F8F2, #50FA7B green, #BD93F9 purple); Catppuccin Mocha #1E1E2E / Latte #EFF1F5 (official light+dark pairing). Polished-reference rule: near-monochrome base + exactly one accent (terminal.shop uses stark black/white + Terminal orange ~#FF5C00).
- Design references: terminal.shop (www.terminal.shop — SSH coffee shop built on Charm/Bubble Tea, stark B&W, razor-contrast, single orange accent; write-up at charm.land/blog/terminaldotshop); SRCL / sacred computer (sacred.computer, github.com/internet-development/www-sacred — open-source React terminal-aesthetic component library whose core discipline is exact monospace character-width and line-height rhythm: every border, card, accordion, action bar aligns to the character grid; MS-DOS-inspired cards; steal its component inventory and spacing math for SwiftUI); Berkeley Graphics / U.S. Graphics Company (usgraphics.com — engineering-datasheet aesthetic: dense tables, part-number labeling like 'TX-02', ruled borders, Berkeley Mono everywhere); Charm's lipgloss/bubbletea repos (github.com/charmbracelet) for terminal layout/styling idioms worth transposing.

## Recommendations
- Font strategy: default to the system monospaced font (SF Mono via Font.system(.monospaced) — zero licensing risk, native rendering) and bundle JetBrains Mono or Departure Mono (both OFL) as the 'signature' display font via ATSApplicationFontsPath; do not bundle SF Mono files or Berkeley Mono.
- Build the UI on a character-grid discipline like SRCL: measure one glyph cell (NSFont advancement) and size/pad every box, rule, and progress bar in whole cell units; use box-drawing characters with .lineSpacing(0) for frames and panels.
- Implement spinners/progress as data, not views: port cli-spinners frame arrays to a Swift enum, drive them with TimelineView(.periodic(by: interval)) and a modulo frame index, and render eighth-block (▏-█) progress bars for sub-character smoothness.
- Render ASCII scenes as one multi-line AttributedString-backed Text for anything under ~2000 chars at <=15fps; switch to Canvas with context.resolve/draw only for 60fps full-surface animations; add .drawingGroup() before applying glow.
- For CRT flavor, start with the free tricks (stacked shadows for glow, radial-gradient vignette, subtle TimelineView flicker), then add a .colorEffect Metal scanline shader (crib Inferno's Interlace, MIT) — keep strength low (5-15% dimming) so it reads as texture, not gimmick; gate effects behind a user toggle for accessibility.
- Theming: three-state (system/light/dark) @AppStorage enum applied via NSApp.appearance (nil = auto/system); define the palette as semantic asset-catalog colors with Any/Dark variants; pick a palette designed as a light/dark pair (Solarized or Catppuccin Latte/Mocha) or near-monochrome + single accent (terminal.shop style) so both modes stay coherent.
- Study sacred.computer and usgraphics.com in a browser with devtools before designing: copy their line-height ratios, border weights, and information density rather than inventing from scratch.
